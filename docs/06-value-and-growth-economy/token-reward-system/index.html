Token Reward System

Non-speculative recognition system for user and creator contributions. Tokens represent value exchange, not investment.

Overview

The Token Reward System is a non-speculative recognition mechanism that acknowledges user and creator contributions to the waQup ecosystem. Unlike credits (which are purchased), tokens are earned through meaningful engagement and contribution.

Core Principle: Tokens recognize value creation, not consumption. They are a form of acknowledgment, not currency.

Key Distinction: Tokens are separate from credits. Credits are purchased and consumed for creation. Tokens are earned and represent recognition of contribution.

Token Types

Practice Tokens

Purpose: Recognize regular, meaningful engagement with created rituals.

  • Earned through: Consistent practice, completion of ritual sequences
  • Not earned through: Mindless scrolling, passive listening
  • Value: Recognition of commitment to personal growth

Creator Tokens

Purpose: Recognize creators who publish high-quality packs in the marketplace.

  • Earned through: Publishing quality packs, positive user feedback, pack sales
  • Value: Recognition of contribution to the ecosystem
  • Note: Separate from revenue (creators receive payment, tokens are recognition)

Referral Tokens

Purpose: Recognize users who bring others to waQup through word-of-mouth.

  • Earned through: Successful referrals (when referred user creates first ritual)
  • Value: Recognition of community building
  • Note: Not a commission system, purely recognition

Feedback Tokens

Purpose: Recognize users who provide constructive feedback that improves the platform.

  • Earned through: Detailed feedback, bug reports, feature suggestions
  • Value: Recognition of contribution to platform improvement

Token Properties

Non-Speculative

  • Not an investment: Tokens have no monetary value
  • Not tradeable: Tokens cannot be sold or exchanged
  • Not convertible: Tokens cannot be converted to credits
  • Purpose: Recognition only, not currency

Recognition Only

  • Visual acknowledgment: Tokens appear in user profile
  • No privileges: Tokens don't unlock features or content
  • No pressure: No leaderboards or comparisons
  • Personal value: Recognition of contribution, not competition

Anti-Gamification Safeguards

  • No streaks: Tokens not earned through daily login
  • No pressure: No "earn more tokens" messaging
  • No manipulation: Tokens are passive recognition
  • No FOMO: No time-limited token earning

Token Distribution

Distribution Principles

  • Meaningful contribution: Tokens recognize genuine value creation
  • No manipulation: Distribution is transparent and fair
  • No pressure: Users don't need to "earn" tokens
  • Recognition, not reward: Tokens acknowledge contribution, not incentivize behavior

Distribution Mechanics

Note: Detailed distribution rules and mechanics to be defined during implementation. The system will be designed to recognize meaningful contribution without creating gamification pressure.

Token Display

User Profile

  • Token count: Visible but not prominent
  • Token types: Breakdown by type (practice, creator, referral, feedback)
  • No comparison: No leaderboards or social comparison
  • Personal value: Recognition of individual contribution

UI Principles

  • Calm display: Tokens shown without pressure or urgency
  • No notifications: No "You earned a token!" popups
  • Passive recognition: Tokens appear naturally, not announced
  • No manipulation: UI doesn't encourage token earning
